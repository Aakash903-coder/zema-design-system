# InteractionBlockingOverlay

### InteractionBlockingOverlay is a component used to block user interaction in situations where the system is not prepared for use, such as on screens or modules that do not have adequate responsiveness for mobile devices or in landscape mode.
---
<br />

## When to use:
- When a screen or module does not have a responsive version for mobile devices, ensuring that the user does not interact with broken or non-functional elements.
- When a mobile device is in landscape mode and the application layout is not adapted to that orientation.
- In temporary situations, where a specific functionality or content needs to be disabled for maintenance or updating.

<br />

## When not to use:
- When the system already offers a responsive version or a viable alternative for interaction on mobile devices.
- In cases where the application can automatically adjust the layout without the need for blocking.
- If blocking interaction can cause frustration to the user, especially in essential functionalities or critical flows.
